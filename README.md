# Production Control System

Асинхронная backend-система для управления производственными партиями (batches), продуктами и аналитикой.

Проект моделирует реальный производственный процесс: создание партий, учет продукции, агрегацию, аналитику и интеграции через webhooks и фоновые задачи.

---

## Стек технологий

- FastAPI — REST API  
- PostgreSQL — база данных  
- SQLAlchemy (async) — ORM  
- Alembic — миграции  
- Celery — фоновые задачи  
- Redis / RabbitMQ — брокер сообщений  
- MinIO (S3) — файловое хранилище  
- Docker / Docker Compose — контейнеризация  

---

## Функциональность

### Batches
- создание партии
- получение списка и деталей партии
- расчет статистики по партии

### Products
- добавление продуктов
- агрегация по уникальным кодам
- расчет метрик

### Analytics
- сравнение партий
- расчет агрегированных показателей
- производительность (products/hour)

### Import / Export
- импорт данных из CSV/XLSX
- экспорт данных
- защита от загрузки слишком больших файлов

### Webhooks
- регистрация подписок
- события: `batch_created`, `report_generated`
- HMAC-подпись запросов
- retry механизм

### Reports
- генерация отчетов по партиям
- асинхронная обработка через Celery

---

## Архитектура

Используемые подходы:

- Service Layer  
- Repository Pattern  
- Unit of Work  
- Separation of Concerns  

Поток данных:

API → Service → Repository → Database

---

## Архитектурные решения

- Unit of Work управляет транзакциями и консистентностью  
- Репозитории изолируют работу с БД  
- Celery используется для фоновых задач  
- Webhooks реализованы с HMAC-подписью  
- MinIO используется как S3-совместимое хранилище  

---

## Безопасность

- аутентификация через `X-API-Key`
- валидация входных данных
- ограничение размера загружаемых файлов
- защита webhook через HMAC
